How to Use GPX Compare

This tool compares multiple GPX runs on a shared route axis with explicit point selection. The correct workflow is strict: select start for all riders, then end for all riders, then build route.

Quick Start
  1. Upload 2-5 GPX files.
  2. Keep only riders you want in comparison checked in the rider panel.
  3. Click 1) Select starts, then click one start point on each included rider.
  4. Click 2) Select ends, then click one end point on each included rider.
  5. Click 3) Build consensus route (0.5m).
  6. Review chart/table and change compare base rider if needed.

1) Upload and Configure Riders

  • Upload up to five .gpx files.
  • Rider names default from file names (without .gpx).
  • Edit rider name and color in the Riders panel.
  • Use the Compare checkbox to include/exclude riders.

2) Map Controls

  • Basemap switcher: road, topo, satellite, dark.
  • Default point mode is All points for precise selection.
  • Click a point to select it for the active selection phase.
  • Selected point is highlighted, and per-rider status updates in the UI.

3) Reference Route Workflow (Required)

Step A: Select Starts

  • Press 1) Select starts.
  • Click the intended start point on every included rider track.
  • Check status lines until all included riders show start OK.

Step B: Select Ends

  • Press 2) Select ends.
  • Click the intended end point on every included rider track.
  • Check status lines until all included riders show end OK.

Step C: Build Route

  • Press 3) Build consensus route (0.5m).
  • The route is generated by averaging trimmed segments at high spatial resolution.
  • After build, comparisons are normalized to this route.

4) Comparison and Playback

  • Select Compare to rider in the route panel.
  • Use the route progress slider in Playback.
  • Gap chart shows time delta vs route distance.
  • Comparison table shows elapsed, gap, speed delta, and route distance.

5) Tips for Best Accuracy

  • Zoom in and select exact track dots for start/end.
  • Exclude riders with bad GPS quality before route build.
  • If a rider trim changes after build, rebuild the route.
  • Use consistent segment boundaries for fair comparisons.

6) Common Problems

  • Build button disabled: at least one included rider is missing start or end selection.
  • No chart data: route not built, no timestamps, or compare rider not selected.
  • Unexpected gaps: verify start/end points and included riders, then rebuild route.
