How to Use GPX Compare

This tool compares multiple GPX runs on a shared route axis with explicit start/end zones and draggable start/end pins. The workflow is strict: draw start box, draw end box, then build route.

Quick Start
  1. Upload 2-5 GPX files.
  2. Keep only riders you want in comparison checked in the rider panel.
  3. Keep Show only selected segment ON to reduce clutter while trimming.
  4. Click 1) Select starts (draw box) and draw start box on map.
  5. Click 2) Select ends (draw box) and draw end box on map.
  6. Use Navigate button to pan/zoom freely between box draws.
  7. Adjust draggable S (start) and E (end) pins if needed.
  8. Use per-rider Zoom Start/Zoom End and +/- controls for fine point nudging.
  9. Click 3) Build timing comparison.
  10. Review chart/table and change compare base rider if needed.

1) Upload and Configure Riders

  • Upload up to five .gpx files.
  • Rider names default from file names (without .gpx).
  • Edit rider name and color in the Riders panel.
  • Use the Compare checkbox to include/exclude riders.

2) Map Controls

  • Basemap switcher: road, topo, satellite, dark.
  • Default point mode is All points for precision.
  • Show only selected segment hides points outside per-rider S/E trims (recommended while adjusting).
  • Turn it off if you need to inspect full laps or locate alternate anchors.
  • Selection is done by drawing a box while in start/end selection mode.
  • The box center-average creates a guessed pin location from points crossing the box.
  • You can drag S/E pins to fine tune exact start/end location.

3) Reference Route Workflow (Required)

Step A: Select Starts

  • Press 1) Select starts (draw box).
  • Draw a rectangle around the shared start zone.
  • The tool averages all included rider points inside the box and places the red S pin.
  • Drag S pin if needed.

Step B: Select Ends

  • Press 2) Select ends (draw box).
  • Draw a rectangle around the shared end zone.
  • The tool averages all included rider points inside the box and places the dark E pin.
  • Drag E pin if needed.

Step C: Build Route

  • Press 3) Build timing comparison.
  • Each rider is trimmed to nearest points from S/E pin anchors.
  • After build, comparisons run on normalized segment progress and elapsed time.
  • Use Zoom to compare route to frame only the active comparison segment.
  • Use Lock zoom to prevent accidental zoom changes while reviewing.

4) Comparison and Playback

  • Select Compare to rider in the route panel.
  • Use the route progress slider in Playback.
  • Gap chart shows time delta over elapsed-time x-axis (seconds ticks).
  • Comparison table is sorted by current leader (fastest first).
  • Use table time-format buttons to toggle mm:ss.fff and seconds.
  • Comparison table shows elapsed, gap, speed delta, and progress.
  • Rider panel shows live segment time while nudging trim points.
  • Use compact rider controls (S-5/S-1/S+1/S+5 and E-5/E-1/E+1/E+5) for fast adjustment across all riders.
  • If trims change after build, status warns that route needs rebuild.

5) Tips for Best Accuracy

  • Draw tight start/end boxes around the intended corridor for best pin guesses.
  • Exclude riders with bad GPS quality before route build.
  • If you move S/E pins after build, rebuild the route.
  • Use consistent segment boundaries for fair comparisons.

6) Common Problems

  • Build button disabled: at least one included rider is missing start or end detection.
  • No chart data: route not built, no timestamps, or compare rider not selected.
  • Unexpected gaps: verify S/E pin placement and included riders, then rebuild route.
